# 에이전트 팀 일하는 방식 (Convention)

이 문서는 **에이전트가 작업하는 저장소**에서 사용할 수 있는 기본 규칙입니다.  
목적·프레임워크·배포 방식이 달라도, **문서 구조와 읽기/쓰기 규칙**을 맞추면 에이전트가 같은 방식으로 동작합니다.

---

## 기본 원칙

- **GitHub 저장소가 단일 소스**: 이슈, 코드, 문서 모두 저장소에 두고 `read_github_file` / `write_github_file`로 읽고 씁니다.
- **docs/ 아래에 역할별 디렉터리**를 두고, 에이전트가 여기를 참조·갱신합니다.
- 저장소에 `docs/` 구조가 **없으면** 이 규칙을 강제하지 않고, **있으면** 아래 구조에 맞춰 읽고 씁니다.

---

## 권장 디렉터리 구조

```
docs/
├── plan/      # 계획·스프린트·기술 스펙 요약 (매니저가 참조·작성)
├── skill/     # 프로젝트 스택·가이드라인·모범 사례 (매니저·개발·QA 참조)
└── issues/    # 이슈별 요약·스펙·결과 (이슈 번호별 파일, 서로 참조 가능)
```

### docs/plan/

- **용도**: 프로젝트/스프린트 계획, 기술 방향, 스펙 요약.
- **에이전트 사용**:
  - **매니저**: 새 이슈 분석 시 기존 `docs/plan/` 파일이 있으면 읽어 맥락에 반영. 필요 시 스펙 요약을 여기 추가.
  - **개발/QA**: 구현·리뷰 시 참고할 계획이 있으면 읽음.

### docs/skill/

- **용도**: 이 저장소에서 쓰는 스택, 코딩 규칙, 배포 방법, 도메인 용어 등.
- **에이전트 사용**:
  - **매니저**: 기술 스펙 작성 전 `docs/skill/`을 읽고, 프로젝트 규칙에 맞게 스택·경로·컨벤션 제안.
  - **개발**: 구현 전 `docs/skill/`을 읽고, 명시된 스택·규칙 준수.
  - **QA**: 리뷰 시 `docs/skill/`의 모범 사례·규칙을 기준으로 검토.

### docs/issues/

- **용도**: 이슈 번호별로 요약·스펙·결과를 남겨 두고, 다른 이슈나 plan/skill에서 참조.
- **파일 예**: `docs/issues/issue-42.md`, `docs/issues/issue-43.md`
- **에이전트 사용**:
  - **매니저**: 이슈 분석·스펙 정리 후 `docs/issues/issue-{번호}.md`에 요약 작성. 다른 이슈와의 연관이 있으면 해당 파일을 참조·링크.
  - **개발**: 구현 요약이나 변경 파일 목록을 같은 파일에 추가해도 됨.
  - **QA**: 리뷰 요약을 같은 파일에 추가하거나, 관련 이슈 파일을 참조.

---

## 읽기/쓰기 규칙 요약

| 역할   | docs/plan | docs/skill | docs/issues |
|--------|-----------|------------|-------------|
| 매니저 | 읽기·필요 시 쓰기 | 읽기 | issue-{N}.md 쓰기·참조 |
| 개발   | 필요 시 읽기     | 읽기 | 필요 시 읽기·추가 |
| QA     | 필요 시 읽기     | 읽기 | 필요 시 읽기·추가 |

- **없는 경로**: 해당 디렉터리나 파일이 없으면 무시하고, 이슈 댓글만으로 진행해도 됨.
- **참조**: `docs/plan/`, `docs/skill/`, `docs/issues/` 내 문서들을 서로 참조(링크·인용)할 수 있음.

---

## 프로젝트에서 사용하려면

1. 에이전트가 작업할 **대상 저장소**에 위와 같이 `docs/plan/`, `docs/skill/`, `docs/issues/`를 만들고,
2. 필요하면 `docs/skill/`에 해당 프로젝트의 스택·규칙을 적어 두면,
3. 에이전트 팀이 이 구조를 읽고 쓰며 일하는 방식에 맞춰 동작합니다.

이 문서(agent-convention)는 에이전트 팀 코드베이스(TeamWP)에 있으며, 대상 저장소에는 **위 구조만 갖추면** 됩니다. 필요 시 이 규칙을 대상 저장소의 `docs/README.md` 등에 요약해 두어도 됩니다.
